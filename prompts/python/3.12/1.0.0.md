## Python 3.12 Coding Standards

You MUST follow these Python 3.12 coding standards when writing or modifying code:

### Target Version

- Target Python 3.12 specifically.
- Use Python 3.12 features where appropriate.

### Imports

- Remove all unused imports.
- Sort imports in the standard order: standard library, third-party, local.
- Use absolute imports over relative imports.

### Code Style

- Maximum line length: 100 characters.
- Use f-strings for string formatting, not `.format()` or `%` formatting.

### Python 3.12 Features

- Use `match` statements for complex conditionals.
- Use type unions with `|` syntax (e.g., `str | None` instead of `Optional[str]`).
- Use `type` statement for type aliases (PEP 695): `type Vector = list[float]`.
- Use f-string debugging with `=` (e.g., `f"{x=}"` for `x=42`).
- Use `@override` decorator from `typing` for method overrides.

### Type Hints

- Add type hints to all function signatures.
- Use built-in generics (`list[str]`, `dict[str, int]`) instead of `typing.List`, `typing.Dict`.
- Use `from __future__ import annotations` is no longer needed in 3.12.

### Error Handling

- Be specific with exception types. Avoid bare `except:` clauses.
- Use exception groups and `except*` for handling multiple exceptions where appropriate.
- Use context managers (`with` statements) for resource management.

### Naming

- Use `snake_case` for functions and variables.
- Use `PascalCase` for classes.
- Use `SCREAMING_SNAKE_CASE` for constants.
